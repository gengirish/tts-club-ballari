## Email body (plain text)

```
Hi everyone,

Thank you for helping us beta test **Sister Stride** — the web app for **Steel Sisters & Striders, Ballari** (SSS Club). Your feedback before we widen the circle matters a lot.

**What we’re testing**
We want real-world use on phones and desktops: sign-in, the member home, progress and challenges, Couch to 5K, community, safety tools — and newer flows like **Walking to 5K** (programme landing + online registration) and **paid event registration** (screenshot upload, passes, host/admin queues) where your role applies.

**Where to go**
Primary app URL: https://sister-stride.intelliforge.tech/

**Accounts**
We’ve set up shared **beta accounts** (member, coach, host, admin) for structured testing. Use the **Password** tab on the login screen with the email/username and password we’ve shared separately (or via our internal tester doc — please don’t forward passwords in email threads).

**What we need from you**
1. Spend 20–30 minutes on the flows that match you (member vs host vs admin if assigned).
2. Note anything confusing, broken, or slow — especially on **mobile data** if you can.
3. Send bugs in a short format: what you clicked, what you expected, what happened, browser + phone model, rough time (IST). Screenshots or screen recordings help.

**Docs (for volunteers who want detail)**
- Beta checklist and areas: see `docs/BETA_TEST_PLAN.md` in the repo if you have Git access, or we can paste a one-pager in the group.
- Deployed URL reference: `docs/DEPLOYED_URLS.md`

**Important**
- This is a **beta** environment — don’t use it for sensitive personal data you wouldn’t want on a test server.
- Don’t enable automation-only OTP bypass flags on production; testers use normal password login for seeded accounts.

Reply to this email (or message in our agreed WhatsApp group) when you’ve had a first pass — even “no blockers” is useful.

With gratitude,
[Your name]
Steel Sisters & Striders / IntelliForge
```
